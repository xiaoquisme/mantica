# Scheduled Tasks (Cron Jobs for Agents)

## Goal

给 Multica 添加定时任务系统。Agent 的 instructions + skills 通过 `agent_config.yaml` 配置（跟其他 agent 一样），定时任务的创建和管理通过前端 UI 完成。Server 端 scheduler 到时间自动给指定 agent 下发 task。

第一个用例：定期巡检 blocked issues，读评论提取依赖，判断依赖是否解除，自动 unblock。

## Current Context

- `agent_config.yaml` 定义 agent（name, provider, instructions, skills）
- Server 已有 background goroutine 模式（`runRuntimeSweeper`）
- Task 可以不关联 issue（chat task 就是 `issue_id = NULL`）
- Agent prompt 构建在 `daemon/prompt.go`，目前支持 issue task 和 chat task
- 前端用 Zustand store + shadcn 组件

## Design

### Agent 定义在 agent_config.yaml（不改）

Patrol agent 跟 Classifier、BA、DEV 一样，就是一个普通 agent：

```yaml
agents:
  - name: Patrol
    provider: hermes
    instructions: |
      You are a Blocked Issue Patrol agent. Your job is to check blocked
      issues and determine if their blocking dependencies have been resolved.
    skills:
      - Blocked Issue Patrol
```

### 定时任务通过前端 UI 管理

用户在 UI 上：
1. 选择一个 agent（如 Patrol）
2. 配置 cron 表达式（如 `*/30 * * * *`）
3. 写 prompt（agent 执行时的具体指令）
4. 启用/禁用

### 数据模型

新建 `scheduled_task` 表：

```sql
CREATE TABLE scheduled_task (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    workspace_id UUID NOT NULL REFERENCES workspace(id) ON DELETE CASCADE,
    name TEXT NOT NULL,
    agent_id UUID NOT NULL REFERENCES agent(id) ON DELETE CASCADE,
    schedule TEXT NOT NULL,             -- cron 表达式
    prompt TEXT NOT NULL,
    enabled BOOLEAN NOT NULL DEFAULT true,
    last_run_at TIMESTAMPTZ,
    next_run_at TIMESTAMPTZ,
    created_by UUID NOT NULL REFERENCES "user"(id),
    created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    UNIQUE(workspace_id, name)
);
```

`agent_task_queue` 加列：
```sql
ALTER TABLE agent_task_queue ADD COLUMN scheduled_task_id UUID REFERENCES scheduled_task(id) ON DELETE SET NULL;
```

### Config Apply 扩展

不改 config apply。scheduled_task 的 CRUD 通过独立 API 管理。

### API Endpoints

```
GET    /api/scheduled-tasks          — 列表（workspace scoped）
POST   /api/scheduled-tasks          — 创建
PUT    /api/scheduled-tasks/{id}     — 更新
DELETE /api/scheduled-tasks/{id}     — 删除
POST   /api/scheduled-tasks/{id}/run — 立即执行一次（调试用）
```

权限：workspace admin/owner。

请求体示例（创建）：
```json
{
  "name": "Blocked Issue Patrol",
  "agent_id": "uuid-of-patrol-agent",
  "schedule": "*/30 * * * *",
  "prompt": "Check all blocked issues..."
}
```

### Server Scheduler

在 `server/cmd/server/main.go` 启动 scheduler goroutine：

```go
go runScheduler(schedulerCtx, queries, taskService, bus)
```

逻辑：
1. 每 30 秒 tick
2. 查 `next_run_at <= now() AND enabled = true`
3. 对每个到期的 scheduled_task：
   - 查 agent 确认有 runtime 且未 archived
   - 创建 task（`EnqueueScheduledTask`）
   - 更新 `last_run_at` 和 `next_run_at`
4. Graceful shutdown 时 cancel context

### Task 类型扩展

新增第三种 task 模式：
- Scheduled task: `scheduled_task_id` 非空，agent 按 prompt 干活

Daemon 侧：
- `Task` 结构体加 `ScheduledTaskID` 和 `ScheduledPrompt`
- `BuildPrompt` 新增 scheduled 分支
- Claim endpoint 返回 scheduled task 数据

### Blocked Issue Patrol — 第一个用例

1. 在 `agent_config.yaml` 添加 Patrol agent + skill
2. `multica config apply` 创建 agent
3. 用户在前端 UI 创建 scheduled task，选 Patrol agent，配 cron + prompt

## Risks & Open Questions

1. **Cron 库**：用 `robfig/cron/v3`，Go 生态标准。
2. **多实例**：当前单实例，暂不需要分布式锁。
3. **Agent 离线**：跳过并 log warning，下个 tick 再检查。
4. **前端 cron 输入**：提供预设选项（每 15 分钟、每小时、每天）+ 自定义 cron 表达式。
5. **Scheduled task 历史**：task 执行记录在 agent_task_queue，通过 scheduled_task_id 关联查询。
